# Firebase Setup Prerequisites

Steps to complete before the app will build and run. Complete them in order.

---

## 1. Create a Firebase project

1. Go to https://console.firebase.google.com
2. Click **Add project** and follow the prompts
3. Once created, open the project dashboard

---

## 2. Enable authentication methods

In the Firebase Console:

1. Navigate to **Authentication → Sign-in method**
2. Enable **Email/Password**
3. Enable **Google** — set a support email when prompted

---

## 2b. Enable Firestore

Firebase Console → **Build → Firestore Database → Create database** → choose a region → **Start in test mode**.

---

## 3. Install required CLI tools

Run these once on your machine:

```bash
npm install -g firebase-tools
firebase login
dart pub global activate flutterfire_cli
```

---

## 4. Register the Android app and generate config files

From the project root (`Paperless_1/`):

```bash
flutterfire configure
```

- Select your Firebase project when prompted
- Select **Android** as the target platform
- This generates two files that are required at build time:
  - `lib/firebase_options.dart`
  - `android/app/google-services.json`

---

## 5. Register the debug SHA-1 fingerprint (required for Google Sign-In)

Run this to get your debug SHA-1:

```bash
keytool -list -v \
  -keystore ~/.android/debug.keystore \
  -alias androiddebugkey \
  -storepass android \
  -keypass android
```

Then in Firebase Console:

1. Go to **Project Settings → Your apps → Android app**
2. Click **Add fingerprint**
3. Paste the SHA-1 value from the command output

---

## 6. Install Flutter dependencies

From the project root:

```bash
flutter pub get
```

---

## 7. Lock Firestore security rules

The database was created in **test mode** (open read/write). Before going to production, replace the rules in **Firebase Console → Firestore Database → Rules** with:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == uid;
    }
  }
}
```

Click **Publish**. This ensures every user can only read and write their own data.

---

## 8. CI/CD pipeline setup (GitHub Actions)

Three workflows are already committed in `.github/workflows/`. They need secrets and (for releases) a signing keystore to function.

### 8a. Add repository secrets

Go to **GitHub → repository → Settings → Secrets and variables → Actions → New repository secret** and add each of the following:

| Secret name | Value |
|---|---|
| `GOOGLE_SERVICES_JSON` | Full contents of `android/app/google-services.json` |
| `FIREBASE_OPTIONS_DART` | Full contents of `lib/firebase_options.dart` |
| `KEYSTORE_BASE64` | Base64-encoded release keystore (see step 8b) |
| `KEY_ALIAS` | Alias used when the keystore was created |
| `KEY_PASSWORD` | Password for the key |
| `STORE_PASSWORD` | Password for the keystore |

### 8b. Generate a release keystore (first time only)

Run once on your machine. Keep the resulting file **secret and backed up** — losing it means you can never update the app on the Play Store.

```bash
keytool -genkey -v \
  -keystore release.keystore \
  -alias paperless \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000
```

Then base64-encode it for the `KEYSTORE_BASE64` secret:

```bash
# macOS / Linux
base64 -w 0 release.keystore | pbcopy   # copies to clipboard

# Windows (PowerShell)
[Convert]::ToBase64String([IO.File]::ReadAllBytes("release.keystore")) | clip
```

Paste the output as the value of the `KEYSTORE_BASE64` secret.

### 8c. Register the release SHA-1 fingerprint

Google Sign-In requires the release keystore's SHA-1 to be registered in Firebase Console, just like the debug one was in step 5.

```bash
keytool -list -v \
  -keystore release.keystore \
  -alias paperless \
  -storepass <your-store-password>
```

Add the **SHA-1** value to **Firebase Console → Project Settings → Your apps → Android app → Add fingerprint**.

### 8d. Trigger the workflows

| Workflow | Trigger | What it does |
|---|---|---|
| `ci.yml` | Every push / PR | Runs `flutter analyze` + `flutter test` |
| `build.yml` | Push to `master` | Builds debug APK → uploads as artifact |
| `release.yml` | Push a `v*` tag | Builds signed release APK → creates GitHub Release |

To publish a release:

```bash
git tag v1.0.0
git push --tags
```

## Verification checklist

- [ ] `lib/firebase_options.dart` exists (generated by `flutterfire configure`)
- [ ] `android/app/google-services.json` exists (generated by `flutterfire configure`)
- [ ] Debug SHA-1 fingerprint registered in Firebase Console
- [ ] `flutter pub get` runs without errors
- [ ] Cold start with no user → lands on LoginScreen
- [ ] Sign up with email/password → goes to HomeScreen
- [ ] Kill and relaunch → skips LoginScreen (session persisted)
- [ ] Sign in with Google → goes to HomeScreen
- [ ] Forgot password → reset email arrives in inbox
- [ ] Sign out from drawer → returns to LoginScreen
- [ ] Firestore security rules published (step 7)
- [ ] All 6 GitHub Actions secrets added (step 8a)
- [ ] Release keystore generated and backed up (step 8b)
- [ ] Release SHA-1 fingerprint registered in Firebase Console (step 8c)
- [ ] CI workflow passes on a test push
- [ ] `git tag v1.0.0 && git push --tags` triggers a GitHub Release
